# ⌘ DevCMD

> 스택별 개발 명령어 레퍼런스 · 개인 CLI 지식 베이스

자주 쓰는 CLI 명령어를 스택별·태그별로 정리하고, 직접 추가·수정하며 쓰는 개인 명령어 레퍼런스 툴입니다. 서버 없이 단일 HTML 파일로 동작합니다.

---

## 특징

**언제 쓰는지 설명 포함** — 명령어 암기가 아닌 맥락 이해. 모든 카드에 "언제 사용하나요?" 설명이 함께 있습니다.

**직접 추가·수정** — 내 프로젝트·팀 특화 명령어를 직접 입력하고 관리할 수 있습니다.

**스택 + 태그 이중 필터** — 상단 탭으로 스택을 고르고, 좌측 사이드바에서 `#deploy`, `#db` 등 태그로 좁힐 수 있습니다.

**즐겨찾기** — 자주 쓰는 명령어만 별도 뷰로 빠르게 접근합니다.

**서버 불필요** — `localStorage`에만 저장됩니다. 오프라인에서도 완전히 동작합니다.

## 사용법

### 시작하기

별도 설치 없이 `devcmd.html` 파일을 브라우저에서 열면 됩니다.

```bash
open devcmd.html       # macOS
start devcmd.html      # Windows
xdg-open devcmd.html   # Linux
```

### 명령어 찾기

- 상단 **스택 탭** 클릭 → 특정 스택만 표시
- 좌측 **태그 사이드바** 클릭 → `#deploy`, `#db` 등으로 좁히기
- 상단 **검색창** → 명령어 내용, 설명, 태그 동시 검색

### 명령어 추가

우상단 **＋ 명령어 추가** 버튼 → 모달에서 입력 후 저장

| 필드 | 설명 |
|------|------|
| 스택 | 소속 스택 선택 |
| 카테고리 | 스택 내 소그룹 (예: 마이그레이션, 배포) |
| 명령어 제목 | 카드에 표시되는 이름 |
| 언제 사용하나요? | 맥락 설명 (핵심 필드) |
| 명령어 | 실제 CLI 명령어 (멀티라인 가능) |
| 태그 | 클릭으로 선택 |

### 카드 액션

| 버튼 | 동작 |
|------|------|
| ★ | 즐겨찾기 토글 |
| ⎘ | 클립보드 복사 |
| ✎ | 수정 모달 오픈 |
| ✕ | 삭제 (커스텀 명령어만) |

> 내장 명령어는 삭제되지 않습니다. 수정 시 원본을 유지하고 새 항목으로 추가됩니다.

## 데이터 저장

모든 데이터는 브라우저 `localStorage`에 저장됩니다.

| 키 | 내용 |
|----|------|
| `dcmd_custom` | 사용자 추가 명령어 |
| `dcmd_favs` | 즐겨찾기 ID 목록 |
| `dcmd_nextid` | 다음 ID 카운터 |

브라우저 캐시를 지우면 커스텀 명령어와 즐겨찾기가 초기화됩니다. 중요한 명령어는 별도로 백업해두세요 (v1.1에서 JSON 내보내기 지원 예정).
